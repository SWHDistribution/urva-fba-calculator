Rate sheet

Per billed unit (a bundle counts as one billed unit). Bundle and oversize are uplifts added on top of the item's base. Shared across the team.

Wholesale bundle = base + uplift (0.55 + 0.25 = 0.80). OA tool bundle = 1.50 + 0.25 = 1.75. Oversize stacks on top.

Supplier → channel

Each distinct value from the retailer column, tagged Wholesale or OA. Drives the base rate. Anything Unset is excluded from the total and flagged on Run. Changes save for the whole team.

Brand → OA type

The brand decides the row's rate: Wholesale, OA tool, OA shoe, or OA non-tool. Each brand string is matched as a case-insensitive substring of item_name (longest match wins). A row with no brand match falls back to the Suppliers tab; if that's unset too, it's excluded and flagged. Shared across the team.

Match priority is by length, so "New Balance" wins over a stray "New". Keep brand strings specific.

Size tier → oversize

Distinct product-size-tier values from the Fee Preview file. Toggle which count as oversize. Defaults: anything with "oversize", "bulky", or "extra-large" is on.
